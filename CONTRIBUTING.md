# Contributing to Tinkerflow-AI

We welcome contributions! Please read our [governance model](GOVERNANCE.md) first.

## Development Workflow

1. Fork the repo and create a feature branch.
2. Run tests locally: `make test`
3. Ensure governance checks pass (they will run on your PR).
4. Open a pull request with a clear description.

## Pull Request Requirements

- All governance checks must pass.
- At least one maintainer review.
- PR title must follow [Conventional Commits](https://www.conventionalcommits.org/).

For major changes, please open an issue first to discuss.
